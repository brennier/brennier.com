Vimtexer Update (12/03/16)

It's been about a month since my last post, and it seems that I was wrong about Vimtexer no longer needing updates. I found a few bugs that were a little annoying and also I just thought that it could be better. I made it a little more customizable, removed text objects (they just seemed tacked-on, in all honesty), added support for other types of files besides .tex, separated the dictionaries from the actual program, turned off suggested dictionaries by default, and refactored the code in an attempt to remove some bugs that I was finding. Incidentally, not once does the plugin use normal mode now, and I think that this has made the code significantly faster as well. Also, I have some plans in mind on how I can improve the plugin even further. It's a little strange how much you can continually improve just about 100 lines of code. In any case, I doubt I'm going to try to create my own LaTeX-esque language in the near future, even just trying to come up with a formal grammar is difficult. Also, I should be posting more actively over this next month as my school semester ends in two weeks. Until then, though, I should get back to studying for my finals in my graduate classes (Real Analysis and Abstract Algebra).

Vimtexer Stable (11/04/16)

I know that I haven't posted any updates here for while, but I just wanted to announce that my vim plugin for LaTeX, vimtexer, is now, at least in my eyes, stable. After a fair bit of coding and adding some new features here and there, I'm pretty happy with the way that it has turned out, and I don't have any more ideas that I can think of to implement. For the future, I'm considering the possibility of creating a new language for typing/formatting mathematics that'll compile into LaTeX. Why go to this extreme? If I have learned anything from the process of developing vimtexer, its that LaTeX could be typed in a much, much faster manner. Creating a new language from scratch will allow people who use any text editor to type in a more efficient way to create beautiful mathematics. Not only that, but I have a few features that I would like to add, such as the ability to set local aliases, to have a separate namespace for "mathematics" and for "regular typing", and to more easily set custom commands that are more powerful and customizable. In any case, I hope that for the moment, people are as satisfied as I am with the speed improvements that vimtexer brings.

Vimtexer released (09/18/16)

I have recently released the first version of my vimtexer program. As you may imagine, it's a vim plugin that allows to type in LaTeX significantly faster. Specifically, it allows the user to map keywords typed in insert mode into arbitrary keypresses, including the calling of functions or just simply mapping into a different string. Perhaps most importantly, the namespace for these keywords are different for each type of file and also is different depending if one is in mathmode or just typing normal LaTeX. Since variables are always a single character, one can use an abundance of keywords while in math mode. This has helped speed up my LaTeX typing extremely. It's still a little specific to my personal preferences at the moment, but I hope to generalize it soon enough. I hope it helps someone out there! I'll be sure to post updates as I use it more!
